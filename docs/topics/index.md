## 本期导读

这里聚焦那些不适合放进单个产品页、但又值得被优先阅读的深度内容。你可以把这里理解成 Stell Hub 文档站里的“专题编辑部”，它负责把零散的技术事实整理成成体系的判断框架，并作为星际枢纽的前沿观测舱。

## 封面专题

### 为什么企业要自研中间件

这篇文章不是在为“自研中间件”做情绪化辩护，而是试图回答几个更根本的问题：

- 中间件为什么会从事务处理与异构连接层，演进为企业级治理与平台化能力
- 企业为什么会在开源、二开、自研之间反复摇摆
- 不同规模企业在中间件决策上，真正的边界条件分别是什么
- 基础架构团队应该如何通过平台工程创造价值，而不是只扮演控制者

- [阅读专题全文：为什么企业要自研中间件](/topics/middleware-evolution)

## 当前专题子项

### 分布式系统中的一致性挑战及其解决路径

这篇子专题更偏基础理论与工程内核，重点讨论分布式系统为什么一定会遇到一致性问题，以及这些问题如何被论文体系和工业系统逐步解决。它与“为什么企业要自研中间件”形成互补：前者更偏企业决策与平台治理，后者更偏一致性理论、共识协议与基础设施实现。

- 从 FLP、CAP、PACELC 看到理论边界如何限制现实系统
- 从 Viewstamped Replication、Paxos、Raft、Zab、PBFT 看到一致性协议如何演进
- 从 Chubby、ZooKeeper、Spanner、etcd、TiKV、Kafka KRaft 看到工程系统如何把一致性真正做成基础设施

- [阅读专题全文：分布式系统中的一致性挑战及其解决路径](/topics/distributed-consistency)
